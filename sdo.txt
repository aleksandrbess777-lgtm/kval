Вывод всех параметров ядра: sysctl –a 
Двунаправленный буфер обмена для удобства.
 
vm.swappiness — определяет склонность системы к использованию swap. Для ВМ лучше снизить, чтобы избежать излишней подкачки. Значение 10 означает, что система будет стараться держать данные в ОЗУ, а не в swap.
Настройка ядра.
echo 'vm.swappiness = 10' | sudo tee -a /etc/sysctl.conf
vm.vfs_cache_pressure — регулирует кэширование VFS‑объектов (каталогов, inodes). Снижение значения помогает лучше кэшировать данные.
echo 'vm.vfs_cache_pressure = 50' | sudo tee -a /etc/sysctl.conf
Сетевая настройка.
net.core.rmem_max и net.core.wmem_max — увеличивают максимальный размер буферов приёма и отправки. Полезно при интенсивной сетевой нагрузке.
echo 'net.core.rmem_max = 26214400' | sudo tee -a /etc/sysctl.conf
echo 'net.core.wmem_max = 26214400' | sudo tee -a /etc/sysctl.conf
net.ipv4.tcp_timestamps — отключает временные метки TCP, что снижает нагрузку на процессор.
echo 'net.ipv4.tcp_timestamps = 0' | sudo tee -a /etc/sysctl.conf

sysctl net.core.rmem_max
sysctl net.core.wmem_max
sysctl net.ipv4.tcp_timestamps

Если виртуальная машина использует тип сетевого подключения NAT, то первоначально необходимо настроить проброс портов. 
Пробросим порт хоста 2222 (или любой свободный) на гостевой порт виртуальной машины номер 22 (порт ssh по умолчанию).
 
Т.к. в качестве сетевого подключения у виртуальной машины используется NAT, при подключении мы будем указывать петлевой адрес localhost (127.0.0.1) c портом, на который был назначен проброс.

Для настройки виртуального принтера нужно зайти в настройки принтеров.
sudo apt install cups — установка системы печати CUPS (Common UNIX Printing System) из официальных репозиториев.
sudo systemctl start cups — немедленный запуск службы CUPS в текущем сеансе.
sudo systemctl enable cups — добавление службы CUPS в автозагрузку, чтобы она запускалась при каждом включении системы.
sudo apt install printer-driver-cups-pdf — установка драйвера виртуального принтера CUPS-PDF, позволяющего «печатать» документы в PDF-файлы без физического принтера. 
Обоснование выбора операционной системы и программного обеспечения
В качестве основной операционной системы выбрана Ubuntu. Данный выбор обусловлен следующими преимуществами: Ubuntu является бесплатной и стабильной системой с длительным сроком поддержки (LTS), что гарантирует безопасность и своевременные обновления. Система имеет развитую экосистему и менеджер пакетов APT, позволяющий легко устанавливать необходимое программное обеспечение. Ubuntu широко распространена в учебных заведениях и в сфере информационных технологий, что делает её удобной платформой для выполнения лабораторных и экзаменационных заданий.
Для работы с документами и выполнения вычислений выбраны:
Офисный пакет LibreOffice – полнофункциональный аналог Microsoft Office, совместимый с форматами DOCX, XLSX и PPTX, распространяемый бесплатно и входящий в стандартный репозиторий Ubuntu.
Калькулятор GNOME – встроенное приложение, поддерживающее как простые арифметические операции, так и инженерный режим для выполнения сложных вычислений.
Для организации удалённого доступа используется связка PuTTY и OpenSSH Server. OpenSSH Server устанавливается на управляемую машину и обеспечивает зашифрованное соединение по протоколу SSH, что гарантирует безопасность передаваемых данных и защиту от несанкционированного доступа. PuTTY выступает в роли графического SSH-клиента, удобного для подключения с других компьютеров, в том числе под управлением Windows. Данное решение демонстрирует навыки удалённого администрирования в сетях.
Для создания электронных копий документов выбран виртуальный принтер printer-driver-cups-pdf (CUPS-PDF). Он интегрируется в систему печати CUPS и позволяет «печатать» любые документы в формат PDF без использования реального принтера. Это удобно для сохранения отчётов, веб-страниц и результатов работы в цифровом виде.


2. Применение средств защиты компьютерных систем:

Резервное копирование
sudo su - переход в режим админа
tar cvpzf backup.tgz –exclude=/proc –exclude=/lost+found –exclude=/backup.tgz –exclude=/mnt –exclude=/sys –exclude=/web / - команда для резервного копирования
 
Создание образа системы
sudo dd if=/dev/sda bs=8M status=progress | xz -c -T0 > ~/backup.img.xz
 
Создаём группы admini и kvaki
sudo groupadd admini
sudo groupadd kvaki

Создаём пользователя kvakushka
sudo useradd -m -s /bin/bash kvakushka
sudo passwd kvakushka

Добавляем sanya и kvakushka в группу kvaki
sudo usermod -aG kvaki sanya
sudo usermod -aG kvaki kvakushka

Добавляем sanya в группу admini
sudo usermod -aG admini sanya

Создаём папку и настраиваем права
sudo mkdir -p /srv/kvaki_project
sudo chown :kvaki /srv/kvaki_project
sudo chmod 750 /srv/kvaki_project
sudo chown sanya:kvaki /srv/kvaki_project

Проверяем права: sanya может создать файл
sudo -u sanya touch /srv/kvaki_project/test_sanya && echo "sanya: OK"

Проверяем: kvakushka не может создать файл (Permission denied)
sudo -u kvakushka touch /srv/kvaki_project/test_kvakushka 2>&1 | grep -q "Permission denied" && echo "kvakushka: доступ запрещён" 
sudo apt install libpam-pwquality -y
 
 
Создаём правило rsyslog для мониторинга авторизации
echo 'auth,authpriv.* /var/log/auth-monitor.log' | sudo tee /etc/rsyslog.d/99-authmon.conf

Перезапускаем службу rsyslog, чтобы применить настройки
sudo systemctl restart rsyslog

Выполняем любую sudo-команду (ls /root) для генерации записи в логе; вывод подавляем в /dev/null
sudo ls /root > /dev/null

Проверяем содержимое лог-файла — видим успешные записи о сессиях sudo и выполненных командах
sudo tail /var/log/auth-monitor.log

Снимки системы.
sudo apt update && sudo apt install timeshift -y
Обновляет списки пакетов и устанавливает утилиту Timeshift, которая создаёт снимки системы (аналог «точек восстановления» в Windows).
sudo timeshift --create
Запускает создание нового снимка системы.
Timeshift выбрал тип RSYNC (копирует файлы, работает на любой файловой системе), устройство для хранения — /dev/sda2.
Синхронизация файлов заняла 107 секунд. Снимок сохранён с именем 2026-06-08_16-53-45 и тегом ondemand (создан вручную).
 
sudo timeshift --list
Показывает все существующие снимки.
В выводе виден один снимок (Num 0) с указанной датой.
